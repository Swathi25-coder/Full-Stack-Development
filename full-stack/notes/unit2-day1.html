This chapter covers two core building blocks: variables & primitive data types (how we store values) and operators (how we compute with and compare those values).

1. Variables

1.1 What is a variable?

A variable is a named storage location whose value can change while the program runs.

firstname = "Maaike";
x = 2;
firstname = "Edward";
x = 7;

1.2 Declaring variables: let, var, const

let nr1 = 12;
var nr2 = 8;
const PI = 3.14159;
KeywordBehavior
letBlock-scoped, can be reassigned
varFunction/global-scoped, older & more error-prone — avoid in new code
constBlock-scoped, cannot be reassigned
const someConstant = 3;
someConstant = 4; // ❌ throws at runtime
OUTPUT:Uncaught TypeError: Assignment to constant variable.
Recommendation for beginners: use const by default, let when the value will change, and avoid var except when reading legacy code.

1.3 Variable naming conventions

  • Start names with a letter, _, or $ — never a digit.
  • No spaces; use camelCase for multi-word names (e.g. ageOfBuyer).
  • Use descriptive names: age, totalPrice instead of x, y.

2. Primitive Data Types

JavaScript is loosely typed — a value's type is determined at assignment, not declared explicitly. There are seven primitive types: string, number, bigint, boolean, symbol, undefined, null.

2.1 Strings

let singleString = 'Hi there!';
let doubleString = "How are you?";

Be careful with quotes inside strings:

let funActivity = 'Let's learn JavaScript'; // ❌ the ' in Let's ends the string early
let funActivity = "Let's learn JavaScript"; // ✅ double quotes allow an unescaped single quote inside

2.1.1 Template literals and interpolation

let language = "JavaScript";
let message = `Let's learn ${language}`;
console.log(message);
OUTPUT:Let's learn JavaScript

2.1.2 Escape characters

let str3 = "New \nline.";
let str4 = "I'm containing a backslash: \\!";
console.log(str3);
console.log(str4);
OUTPUT:New
line.
I'm containing a backslash: \!

2.2 Numbers

JavaScript has a single numeric type: a 64-bit floating-point number, covering integers, decimals, exponentials, octal, hexadecimal, and binary.

let intNr  = 1;
let decNr  = 1.5;
let expNr  = 1.4e15;
let octNr  = 0o10;   // 8
let hexNr  = 0x3E8;  // 1000
let binNr  = 0b101;  // 5

2.3 Big integers (bigint)

let bigNr = 90071992547409920n;  // BigInt
let intNr = 1;
let result = bigNr + intNr;   // ❌ mixing BigInt and Number
OUTPUT:Uncaught TypeError: Cannot mix BigInt and other types, use explicit conversions.

2.4 Booleans

let bool1 = false;
let bool2 = true;
let objectIsDeleted = false;
let lightIsOn = true;

2.5 Symbols

let str1 = "JavaScript is fun!";
let str2 = "JavaScript is fun!";
console.log("These two strings are the same:", str1 === str2);

let sym1 = Symbol("JavaScript is fun!");
let sym2 = Symbol("JavaScript is fun!");
console.log("These two Symbols are the same:", sym1 === sym2);
OUTPUT:These two strings are the same: true
These two Symbols are the same: false

Strings with the same characters are equal. Symbols are always unique, even with an identical description — often used as unique object keys.

2.6 undefined

let unassigned;
console.log(unassigned);
OUTPUT:undefined

2.7 null

let terribleThingToDo = undefined;
let lastName;
console.log("Same undefined:", lastName === terribleThingToDo);

let betterOption = null;
console.log("Same null:", lastName === betterOption);
OUTPUT:Same undefined: true
Same null: false
typeof null returns "object" — a well-known historical bug kept for backward compatibility.

3. Inspecting and Converting Data Types

3.1 typeof

let str    = "Hello";
let nr     = 7;
let bigNr  = 12345678901234n;
let bool   = true;
let sym    = Symbol("unique");
let undef  = undefined;
let unknown = null;
console.log("str", typeof str);
console.log("nr", typeof nr);
console.log("bigNr", typeof bigNr);
console.log("bool", typeof bool);
console.log("sym", typeof sym);
console.log("undef", typeof undef);
console.log("unknown", typeof unknown);
OUTPUT:str string
nr number
bigNr bigint
bool boolean
sym symbol
undef undefined
unknown object

3.2 Implicit type coercion

let nr1 = 2;
let nr2 = "2";
console.log(nr1 * nr2);   // string coerced to number
console.log(nr1 + nr2);   // + also means string concatenation
OUTPUT:4
22

3.3 Explicit conversions: String(), Number(), Boolean()

let nrToStr = 6;
nrToStr = String(nrToStr);
console.log(nrToStr, typeof nrToStr);
OUTPUT:6 string
let strToNr = "12";
strToNr = Number(strToNr);
console.log(strToNr, typeof strToNr);

let nullToNr = Number(null);
let emptyStrToNr = Number("");
console.log("null", nullToNr, typeof nullToNr);
console.log("empty string", emptyStrToNr, typeof emptyStrToNr);

let strToNr2 = Number("hello");
console.log(strToNr2, typeof strToNr2);
OUTPUT:12 number
null 0 number
empty string 0 number
NaN number

NaN stands for "Not a Number" — its type is still number.

let strToBool = Boolean("any string will return true");
console.log(strToBool, typeof strToBool);

let strToBool2 = Boolean("false");
let emptyStrToBool = Boolean("");
console.log(strToBool2, typeof strToBool2);
console.log(emptyStrToBool, typeof emptyStrToBool);
OUTPUT:true boolean
true boolean
false boolean
Boolean conversion is false only for: null, undefined, 0, NaN, and "" (empty string). Everything else — even the string "false" — converts to true.

4. Arithmetic Operators

Covered: + - * / ** % ++ --

4.1 Addition (+)

let result1 = 12 + 14;
console.log(result1);
let result2 = "Hello " + "addition";
console.log(result2);
OUTPUT:26
Hello addition

4.2 Subtraction (-)

let result1 = 20 - 4;
let result2 = "Hi" - 3;
console.log(result1, result2);
OUTPUT:16 NaN

4.3 Multiplication (*)

let result1 = 15 * 10;
let result2 = "Hi" * 3;
console.log(result1, result2);
OUTPUT:150 NaN

4.4 Division (/)

let result1 = 30 / 5;
console.log(result1);
OUTPUT:6

4.5 Exponentiation (**)

let result1 = 2 ** 3;
console.log(result1);
OUTPUT:8

4.6 Modulus (%)

console.log(`10 % 3 = ${10 % 3}`);
console.log(`8 % 2 = ${8 % 2}`);
console.log(`15 % 4 = ${15 % 4}`);
OUTPUT:10 % 3 = 1
8 % 2 = 0
15 % 4 = 3

4.7 Increment and decrement

let nr1 = 4;
nr1++;
console.log(nr1);

let nr2 = 4;
nr2--;
console.log(nr2);
OUTPUT:5
3

Prefix vs postfix

let nr = 2;
console.log(nr++); // returns OLD value, then increments
console.log(nr);

let nr_b = 2;
console.log(++nr_b); // increments FIRST, then returns
OUTPUT:2
3
3
let nr1 = 4, nr2 = 5, nr3 = 2;
console.log(nr1++ + ++nr2 * nr3++);
// ++nr2 -> 6 ; nr3++ uses 2 (then becomes 3) ; 6*2=12
// nr1++ uses 4 (then becomes 5) ; 4+12=16
OUTPUT:16

4.8 Operator precedence

let result = 3 + 4 * 2 / 8;
console.log(result); // 4*2=8, 8/8=1, 3+1=4
OUTPUT:4

5. Assignment Operators

let x = 2;
x += 2;   // 4
x -= 2;   // 2
x *= 6;   // 12
x /= 3;   // 4
x **= 2;  // 16
x %= 3;   // 1
console.log(x);
OUTPUT:1

6. Comparison Operators

6.1 Equality: == vs ===

let x = 5;
let y = "5";
console.log(x == y);
console.log(x === y);
OUTPUT:true
false
Best practice: always prefer === and !== over == and !=.

6.2 Relational operators

let x = 5, y = 6;
console.log(y > x, x > y, y > y, y >= y);
console.log(y < x, x < y, y < y, y <= y);
OUTPUT:true false false true
false true false true

7. Logical Operators

let x = 1, y = 2, z = 3;

console.log(x < y && y < z); // AND
console.log(x > y && y < z);

console.log(x > y || y < z); // OR
console.log(x > y || y > z);

let flag = false;
console.log(!flag);          // NOT
OUTPUT:true
false
true
false
true

8. Small Integrated Example

let miles = 130;
const MILES_TO_KM = 1.60934;
let km = miles * MILES_TO_KM;
let message = `The distance of ${km} kms is equal to ${miles} miles`;
let isLongTrip = km > 200 && km < 500;
console.log(message);
console.log("Is this a long trip?", isLongTrip);
OUTPUT:The distance of 209.2142 kms is equal to 130 miles
Is this a long trip? true

This ties together let/const, number primitives, arithmetic (*), template literals, and comparison + logical operators (>, <, &&) in one program.
